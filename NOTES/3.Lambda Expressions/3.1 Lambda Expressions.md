# Lambda Expressions in Java

## Introduction

Lambda Expressions were introduced in Java 8 to support Functional Programming.

A Lambda Expression is simply an **anonymous function**, which means:

* It has no name.
* It can have parameters.
* It can have a return value.
* It can be passed as an argument to methods.
* It can be assigned to a variable of a Functional Interface type.

Lambda Expressions help reduce boilerplate code and make Java programs more concise and readable.

---

# What is a Lambda Expression?

A Lambda Expression is a short block of code that can be passed around and executed later.

### Traditional Method

```java id="j5v3mz"
public int add(int a, int b) {
    return a + b;
}
```

### Lambda Equivalent

```java id="opnjlwm"
(a, b) -> a + b
```

Notice that the Lambda Expression has:

* No method name
* No return type
* No access modifier

This is why it is called an anonymous function.

---

# Why Lambda Expressions?

Before Java 8, implementing interfaces often required a lot of boilerplate code.

### Without Lambda

```java id="l2s4nv"
Runnable runnable = new Runnable() {

    @Override
    public void run() {
        System.out.println("Running...");
    }
};
```

### With Lambda

```java id="3l2dlc"
Runnable runnable =
        () -> System.out.println("Running...");
```

The Lambda version is shorter and easier to read.

---

# Lambda Expression Syntax

A Lambda Expression consists of three parts:

```java id="0ycj9j"
(parameters) -> {
    body
}
```

### Components

#### 1. Parameters

Input values passed to the Lambda Expression.

```java id="g7v91o"
(name)
```

```java id="a3q93u"
(a, b)
```

---

#### 2. Arrow Token

The arrow symbol connects parameters with the body.

```java id="p93v5k"
->
```

---

#### 3. Body

Contains the logic to execute.

```java id="jrt9ny"
(name) -> {
    System.out.println(name);
}
```

---

# Lambda Syntax Examples

## No Parameters

```java id="1h3i09"
() -> System.out.println("Hello");
```

---

## One Parameter

```java id="kl3v21"
name -> System.out.println(name);
```

---

## Multiple Parameters

```java id="ah6s42"
(a, b) -> a + b;
```

---

# Single Statement Lambda

If the body contains only one statement, curly braces are optional.

### With Curly Braces

```java id="l31m67"
name -> {
    System.out.println(name);
}
```

### Without Curly Braces

```java id="5o2rw1"
name -> System.out.println(name);
```

---

# Multiple Statements Lambda

If the body contains multiple statements, curly braces are required.

```java id="t8r6w0"
name -> {

    System.out.println("Processing...");

    System.out.println(name);
};
```

# Lambda Expressions and Functional Interfaces

The primary purpose of Lambda Expressions is to provide implementations for Functional Interfaces.

A Functional Interface contains exactly one abstract method.

### Example Functional Interface

```java id="u6d4tz"
@FunctionalInterface
interface Printable {

    void print(String message);
}
```

### Lambda Implementation

```java id="j9r0cp"
Printable printable =
        message -> System.out.println(message);
```

---

# Complete Example

```java id="j7n1xu"
@FunctionalInterface
interface Printable {

    void print(String message);
}

public class LambdaExpressionDemo {

    public static void main(String[] args) {

        Printable printable =
                message -> System.out.println(message);

        printable.print("Hello Lambda Expression");
    }
}
```

### Output

```text id="dr39vf"
Hello Lambda Expression
```

# Assigning Lambda Expressions to Variables

A Lambda Expression can be assigned to a variable whose type is a Functional Interface.

```java id="e4n0sq"
Printable printable =
        message -> System.out.println(message);
```

Here:

```text id="84zihv"
printable → Variable

Printable → Functional Interface

message -> System.out.println(message)
           → Lambda Expression
```

# Lambda Expression Simplification

## Step 1

```java id="lkv7yo"
(String message) -> {
    System.out.println(message);
}
```

---

## Step 2

Remove parameter type.

```java id="dhk60x"
(message) -> {
    System.out.println(message);
}
```

---

## Step 3

Remove curly braces for a single statement.

```java id="bhn9gs"
(message) ->
    System.out.println(message);
```

---

## Step 4

Remove parentheses for a single parameter.

```java id="nuhgbt"
message ->
    System.out.println(message);
```

This is the most concise form.

# Common Built-in Functional Interfaces

Java provides several predefined Functional Interfaces in the `java.util.function` package.

| Interface         | Description                       |
| ----------------- | --------------------------------- |
| Predicate<T>      | Returns boolean                   |
| Function<T,R>     | Accepts input and returns output  |
| Consumer<T>       | Accepts input and returns nothing |
| Supplier<T>       | Returns a value without input     |
| BiPredicate<T,U>  | Two inputs, boolean output        |
| BiFunction<T,U,R> | Two inputs, returns output        |
| BiConsumer<T,U>   | Two inputs, no return value       |

# Advantages of Lambda Expressions

* Less boilerplate code
* Improved readability
* Supports Functional Programming
* Easy implementation of Functional Interfaces
* Works seamlessly with Stream API
* Enables passing behavior as data

---

# Limitations

* Can only be used with Functional Interfaces.
* Complex logic may reduce readability.
* Debugging can sometimes be harder than traditional methods.

---

# Key Points to Remember

1. Lambda Expressions were introduced in Java 8.

3. It has no name and no return type declaration.

5. The primary purpose of Lambda Expressions is to implement Functional Interfaces.
6. A Functional Interface contains exactly one abstract method.
7. Lambda syntax consists of:

# Summary

Lambda Expressions are one of the most important features introduced in Java 8. They provide a concise way to represent anonymous functions and are primarily used to implement Functional Interfaces. Lambda Expressions simplify code, improve readability, and form the foundation of Functional Programming in Java.
