# Functional Interfaces in Java

## Introduction

Functional Interfaces were introduced in Java 8 and are one of the core concepts behind Functional Programming in Java.

A Functional Interface is an interface that contains **exactly one abstract method**. It can also contain any number of **default methods** and **static methods**.

Functional Interfaces are primarily used with **Lambda Expressions**, allowing developers to write concise and readable code.

---

# What is a Functional Interface?

A Functional Interface is an interface that contains:

* Exactly one abstract method
* Zero or more default methods
* Zero or more static methods

### Example

```java
@FunctionalInterface
interface MyFunctionalInterface {

    void execute(); // Abstract Method

    default void m1() {
        System.out.println("Default Method");
    }

    static void m2() {
        System.out.println("Static Method");
    }
}
```

---

# @FunctionalInterface Annotation

Java provides the `@FunctionalInterface` annotation to explicitly declare an interface as a Functional Interface.

Although this annotation is optional, it is considered a best practice because it allows the compiler to verify that the interface contains only one abstract method.

### Example

```java
@FunctionalInterface
interface Printable {
    void print(String message);
}
```

### Invalid Example

```java
@FunctionalInterface
interface Printable {

    void print(String message);

    void display(String message);
}
```

The compiler will generate an error because the interface contains more than one abstract method.

---

# Rules of Functional Interfaces

A Functional Interface:

✅ Must contain exactly one abstract method

✅ Can contain any number of default methods

✅ Can contain any number of static methods

✅ Can be implemented using Lambda Expressions

❌ Cannot contain multiple abstract methods

# Complete Example

```java
package functionalinterfaces;

@FunctionalInterface
interface Printable {

    void print(String message);

    default void m1() {
        System.out.println("Default Method M1");
    }

    static void m2() {
        System.out.println("Static Method M2");
    }
}

public class FunctionalInterfaceDemo {

    public static void main(String[] args) {

        Printable printable =
                message -> System.out.println(message);

        printable.print("Hello Functional Interface");

        printable.m1();

        Printable.m2();
    }
}
```

### Output

```text
Hello Functional Interface
Default Method M1
Static Method M2
```

# Lambda Expression Simplification

### Version 1

```java
Printable printable = (String message) -> {
    System.out.println(message);
};
```

### Version 2

```java
Printable printable = (message) -> {
    System.out.println(message);
};
```

### Version 3

```java
Printable printable = (message) ->
        System.out.println(message);
```

### Version 4

```java
Printable printable =
        message -> System.out.println(message);
```

This is the most concise form.

# Built-in Functional Interfaces

Java 8 introduced the `java.util.function` package, which contains several predefined Functional Interfaces.

Some commonly used Functional Interfaces include:

| Functional Interface | Description                       |
| -------------------- | --------------------------------- |
| Predicate<T>         | Accepts input and returns boolean |
| Function<T,R>        | Accepts input and returns output  |
| Consumer<T>          | Accepts input and returns nothing |
| Supplier<T>          | Supplies a value without input    |
| UnaryOperator<T>     | Same input and output type        |
| BinaryOperator<T>    | Two inputs with same output type  |
| BiPredicate<T,U>     | Two inputs and returns boolean    |
| BiFunction<T,U,R>    | Two inputs and returns output     |
| BiConsumer<T,U>      | Two inputs and no return value    |

# Why Functional Interfaces?

Functional Interfaces provide:

* Cleaner code
* Less boilerplate code
* Better readability
* Easy integration with Lambda Expressions
* Foundation for Stream API
* Support for Functional Programming in Java

---

# Key Points

* Functional Interfaces were introduced in Java 8.
* A Functional Interface contains exactly one abstract method.
* It may contain multiple default methods.
* It may contain multiple static methods.
* `@FunctionalInterface` annotation is optional but recommended.
* Lambda Expressions are used to implement Functional Interfaces.
* Java provides many predefined Functional Interfaces in the `java.util.function` package.
* Functional Interfaces are the foundation of modern Java Functional Programming.

---

# Summary

A Functional Interface is a special type of interface that contains only one abstract method. It enables the use of Lambda Expressions and plays a vital role in Java Functional Programming. Java provides both custom and predefined Functional Interfaces, making code more concise, readable, and maintainable.
