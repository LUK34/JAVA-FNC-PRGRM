# Higher-Order Functions in Java

## Introduction

Higher-Order Functions are one of the most powerful concepts in Functional Programming.

A Higher-Order Function is a function that:

1. Takes one or more functions as arguments.
2. Returns a function as its result.

Since Java 8 introduced:

* Lambda Expressions
* Functional Interfaces
* Method References
* Stream API

Java developers can now implement Higher-Order Functions easily.

---

# What is a Higher-Order Function?

A Higher-Order Function is a function that:

✅ Accepts another function as a parameter

OR

✅ Returns another function

If either condition is true, it is considered a Higher-Order Function.

---

# Definition

```text
A Higher-Order Function is a function that
takes another function as an argument
or returns another function as a result.
```

---

# Relationship with First-Class Functions

Before understanding Higher-Order Functions, we must understand First-Class Functions.

Because functions can:

* Be assigned to variables
* Be passed as arguments
* Be returned from methods

we can create Higher-Order Functions.

Without First-Class Functions, Higher-Order Functions would not be possible.

---

# Higher-Order Function Example #1

## Function as an Argument

```java
import java.util.function.Function;

public class HigherOrderExample
{
    public static void applyFunction(
            Function<Integer,Integer> function,
            Integer value)
    {
        System.out.println(
                function.apply(value));
    }

    public static void main(String[] args)
    {
        Function<Integer,Integer> square =
                number -> number * number;

        applyFunction(square, 5);
    }
}
```

Output:

```text
25
```

---

# Why is it a Higher-Order Function?

This method:

```java
applyFunction()
```

accepts another function:

```java
Function<Integer,Integer>
```

as a parameter.

Therefore:

```text
applyFunction()
```

is a Higher-Order Function.

---

# Visual Representation

```text
square
   |
   v
applyFunction(square, 5)
   |
   v
25
```

# Higher-Order Function Example #2

## Returning a Function

```java
import java.util.function.Function;

public class HigherOrderReturnExample
{
    public static Function<Integer,Integer>
    createMultiplier(int multiplier)
    {
        return number -> number * multiplier;
    }

    public static void main(String[] args)
    {
        Function<Integer,Integer> triple =
                createMultiplier(3);

        System.out.println(
                triple.apply(5));
    }
}
```

Output:

```text
15
```

---

# Why is it a Higher-Order Function?

The method:

```java
createMultiplier()
```

returns another function.

```java
return number -> number * multiplier;
```

Therefore:

```text
createMultiplier()
```

is a Higher-Order Function.

---

# Visual Representation

```text
createMultiplier(3)
        |
        v
number -> number * 3
        |
        v
triple.apply(5)
        |
        v
15
```

# Real-World Example

Suppose we have a list:

```java
List<Integer> numbers =
        Arrays.asList(1,2,3,4,5);
```

We want to:

* Multiply by 2
* Multiply by 3
* Multiply by 10

Instead of creating multiple methods:

```java
doubleNumbers()
tripleNumbers()
tenTimesNumbers()
```

we can create a reusable Higher-Order Function.

---

# Example

```java
import java.util.List;
import java.util.function.Function;
import java.util.stream.Collectors;

public class HigherOrderStreamExample
{
    public static List<Integer>
    applyFunctionToList(
            List<Integer> numbers,
            Function<Integer,Integer> function)
    {
        return numbers.stream()
                      .map(function)
                      .collect(Collectors.toList());
    }

    public static void main(String[] args)
    {
        List<Integer> numbers =
                List.of(1,2,3,4,5);

        Function<Integer,Integer> multiplyByTwo =
                number -> number * 2;

        List<Integer> result =
                applyFunctionToList(
                        numbers,
                        multiplyByTwo);

        System.out.println(result);
    }
}
```

Output:

```text
[2, 4, 6, 8, 10]
```

---

# Dynamic Behavior

Now suppose the requirement changes.

Instead of:

```java
number * 2
```

we need:

```java
number * 3
```

We simply change the Lambda Expression.

```java
Function<Integer,Integer> multiplyByThree =
        number -> number * 3;
```

and reuse the same method.

```java
applyFunctionToList(
        numbers,
        multiplyByThree);
```

Output:

```text
[3, 6, 9, 12, 15]
```

Notice:

```text
No changes to applyFunctionToList()
```

This is the power of Higher-Order Functions.

---

# How Streams Use Higher-Order Functions

Consider:

```java
numbers.stream()
       .map(number -> number * 2)
       .collect(Collectors.toList());
```

The method:

```java
map()
```

accepts another function:

```java
number -> number * 2
```

Therefore:

```text
map()
```

is a Higher-Order Function.

---

# More Examples in Stream API

## filter()

```java
numbers.stream()
       .filter(number -> number % 2 == 0)
       .toList();
```

The function:

```java
number -> number % 2 == 0
```

is passed as an argument.

Therefore:

```text
filter()
```

is a Higher-Order Function.

---

## sorted()

```java
employees.stream()
         .sorted(
             (e1, e2) ->
                 e1.getName()
                   .compareTo(e2.getName()))
         .toList();
```

The comparator function is passed as an argument.

Therefore:

```text
sorted()
```

is also a Higher-Order Function.

# Benefits of Higher-Order Functions

## 1. Code Reusability

Write the logic once.

Pass different functions.

Example:

```java
applyFunctionToList()
```

can be reused for:

* Square
* Double
* Triple
* Cube

---

## 2. Cleaner Code

Functional Programming reduces:

* for loops
* while loops
* if statements

This makes code easier to read.

---

## 3. Flexibility

Behavior can be changed dynamically.

Example:

```java
multiplyByTwo
multiplyByThree
multiplyByTen
```

without changing the original method.

---

## 4. Easy Testing

Higher-Order Functions usually:

* Avoid shared state
* Avoid side effects

which makes testing easier.

---

## 5. Works Great with Functional Programming

Higher-Order Functions are heavily used with:

* Lambda Expressions
* Functional Interfaces
* Stream API
* Method References

---

# When Should You Use Higher-Order Functions?

Use Higher-Order Functions when:

* Writing reusable code
* Working with Streams
* Using Lambda Expressions
* Building flexible APIs
* Implementing callbacks
* Performing data transformations

---

# Higher-Order Functions vs Normal Functions

| Feature          | Normal Function | Higher-Order Function |
| ---------------- | --------------- | --------------------- |
| Accepts Data     | Yes             | Yes                   |
| Accepts Function | No              | Yes                   |
| Returns Function | No              | Yes                   |
| Reusable         | Limited         | High                  |
| Flexibility      | Low             | High                  |

---

# Functional Programming Connection

Higher-Order Functions are built on:

```text
First-Class Functions
        ↓
Higher-Order Functions
        ↓
Streams
        ↓
Functional Programming
```

This is why understanding First-Class Functions is important before learning Streams.

* A Higher-Order Function is a function that:

  * Accepts another function as an argument
  * Returns another function as a result

  * Lambda Expressions
  * Functional Interfaces
  * Method References
  * Stream API
* Common examples:

  * map()
