# Introduction to Functional Programming in Java

## What is Functional Programming?

Functional Programming is a programming style that focuses on **what to do** rather than **how to do it**.

Before Java 8, Java developers mainly used **Imperative Programming**, where they wrote step-by-step instructions to solve a problem.

Starting from Java 8, Java introduced Functional Programming features such as:

* Lambda Expressions
* Stream API
* Method References
* Functional Interfaces

These features allow developers to write cleaner, shorter, and more expressive code.

---

# Imperative Programming

In Imperative Programming, we explicitly tell Java **how** to perform a task.

We use:

* Variables
* Loops
* Conditional Statements
* Step-by-Step Instructions

### Example: Sum of Even Numbers

```java
public int calculateSum(List<Integer> numbers)
{
    int sum = 0;

    for(Integer number : numbers)
    {
        if(number % 2 == 0)
        {
            sum += number;
        }
    }

    return sum;
}
```

### How it Works

1. Create a variable called `sum`.
2. Iterate through each number.
3. Check whether the number is even.
4. Add the even number to `sum`.
5. Return the result.

### Characteristics of Imperative Programming

* Uses loops (`for`, `while`, `do-while`)
* Uses conditional statements (`if`, `if-else`)
* Modifies variables
* Focuses on **how** a task should be performed
* Developer controls every step

---

# Functional Programming

In Functional Programming, we focus on **what** needs to be done.

Instead of manually writing loops and conditions, we use functions and stream operations.

### Example: Sum of Even Numbers

```java
public int calculateSum(List<Integer> numbers)
{
    return numbers.stream()
                  .filter(number -> number % 2 == 0)
                  .mapToInt(Integer::intValue)
                  .sum();
}
```

### How it Works

1. Convert the list into a Stream.
2. Filter only even numbers.
3. Convert wrapper objects to primitive integers.
4. Calculate the sum.

Notice that we never:

* Create a variable
* Write a loop
* Write an `if` statement

We simply describe the operations we want.

# Understanding the Stream Pipeline

Consider:

```java
numbers.stream()
       .filter(number -> number % 2 == 0)
       .mapToInt(Integer::intValue)
       .sum();
```

### Step 1: stream()

Converts a collection into a stream.

```java
numbers.stream()
```

---

### Step 2: filter()

Keeps only the elements that match the condition.

```java
.filter(number -> number % 2 == 0)
```

Example:

```text
Input:
[1, 2, 3, 4, 5, 6]

Output:
[2, 4, 6]
```

---

### Step 3: mapToInt()

Converts Integer objects to primitive int values.

```java
.mapToInt(Integer::intValue)
```

---

### Step 4: sum()

Calculates the sum.

```java
.sum()
```

Result:

```text
2 + 4 + 6 = 12
```

# Why Functional Programming?

Functional Programming provides several advantages.

---

## 1. Less Code

### Imperative

```java
int sum = 0;

for(Integer number : numbers)
{
    if(number % 2 == 0)
    {
        sum += number;
    }
}
```

### Functional

```java
numbers.stream()
       .filter(number -> number % 2 == 0)
       .mapToInt(Integer::intValue)
       .sum();
```

The Functional version is shorter and easier to read.

---

## 2. Better Readability

The code clearly describes the operations being performed.

```java
filter()
mapToInt()
sum()
```

A reader can quickly understand the logic.

---

## 3. No State Changes

Imperative Programming modifies variables.

```java
sum += number;
```

The value of `sum` changes repeatedly.

Functional Programming avoids mutable state whenever possible.

```java
numbers.stream()
       .filter(...)
       .sum();
```

No variable is being modified.

---

## 4. Easier Debugging

Since there are fewer state changes:

* Less chance of bugs
* Easier troubleshooting
* More predictable behavior

---

## 5. Supports Parallel Processing

Streams can easily run in parallel.

```java
numbers.parallelStream()
       .filter(number -> number % 2 == 0)
       .mapToInt(Integer::intValue)
       .sum();
```

Multiple threads can process data simultaneously.

This improves performance for large datasets.

# Comparison Table

| Feature             | Imperative Programming | Functional Programming |
| ------------------- | ---------------------- | ---------------------- |
| Focus               | How to do it           | What to do             |
| Loops               | Yes                    | No                     |
| If Statements       | Yes                    | Rarely                 |
| Variables           | Frequently Modified    | Usually Avoided        |
| Code Length         | More                   | Less                   |
| Readability         | Moderate               | High                   |
| Parallel Processing | Difficult              | Easy                   |
| Debugging           | More Complex           | Simpler                |

---

# Summary

* Functional Programming was introduced in Java 8.
* It focuses on **what to do** rather than **how to do it**.
* Imperative Programming uses loops, variables, and conditional statements.
* Functional Programming uses:

  * Lambda Expressions
  * Stream API
  * Method References
  * Functional Interfaces
* Functional code is:

  * Shorter
  * Cleaner
  * Easier to read
  * Easier to debug
  * Better suited for parallel processing

Functional Programming is one of the most important features introduced in Java 8 and is heavily used in modern Java applications.
