# Functions as First-Class Objects in Java

## Introduction

One of the most important concepts in Functional Programming is **Functions as First-Class Objects**.

A first-class function means that a function can be treated like any other value.

Just as we can:

* Store numbers in variables
* Store strings in variables

we can also:

* Store functions in variables
* Pass functions as arguments
* Return functions from other functions

Java achieves this through:

* Lambda Expressions
* Functional Interfaces

Introduced in Java 8.

---

# What Does "First-Class Function" Mean?

A function is considered a first-class object if it can:

✅ Be assigned to a variable

✅ Be passed as an argument to another function

✅ Be returned from another function

---

## Real-Life Example

Imagine a calculator.

Instead of hard-coding operations:

```java
add()
subtract()
multiply()
divide()
```

we can pass the operation itself as a parameter.

This makes our code more flexible and reusable.

---

# 1. Assigning a Function to a Variable

Before Java 8, methods could not be stored directly in variables.

Java 8 introduced Lambda Expressions.

---

## Example

```java
import java.util.function.Function;

public class FirstClassFunctionDemo
{
    public static void main(String[] args)
    {
        Function<Integer, Integer> square =
                number -> number * number;

        System.out.println(square.apply(5));
    }
}
```

Output:

```text
25
```

---

## Understanding the Code

### Step 1

Create a variable of type `Function`.

```java
Function<Integer, Integer> square
```

Meaning:

```text
Input  -> Integer
Output -> Integer
```

---

### Step 2

Assign a Lambda Expression.

```java
number -> number * number
```

Equivalent to:

```java
public Integer square(Integer number)
{
    return number * number;
}
```

---

### Step 3

Call the function.

```java
square.apply(5);
```

Output:

```text
25
```

---

# Why Is This a First-Class Function?

Because the function is stored in a variable:

```java
Function<Integer, Integer> square
```

Just like:

```java
int age = 25;
String name = "Luke";
```

---

# Pure Function Example

```java
Function<Integer, Integer> square =
        number -> number * number;
```

This is a Pure Function because:

* Same input → Same output
* No side effects
* No external state changes

Example:

```java
square.apply(5);
square.apply(5);
square.apply(5);
```

Output:

```text
25
25
25
```

# 2. Passing a Function as an Argument

One function can accept another function as a parameter.

This is one of the biggest advantages of Functional Programming.

---

## Example

```java
import java.util.function.Function;

public class FunctionArgumentExample
{
    public static void applyFunction(
            Function<Integer,Integer> function,
            Integer value)
    {
        System.out.println(
                function.apply(value));
    }

    public static void main(String[] args)
    {
        Function<Integer,Integer> doubleValue =
                number -> number * 2;

        applyFunction(doubleValue, 10);
    }
}
```

Output:

```text
20
```

---

## How It Works

### Lambda Function

```java
number -> number * 2
```

Stored inside:

```java
doubleValue
```

---

### Pass Function as Argument

```java
applyFunction(doubleValue, 10);
```

Here we are passing:

* Function
* Data

to another method.

---

### Execute Function

```java
function.apply(value);
```

Output:

```text
20
```

---

# Benefits

Instead of writing:

```java
doubleValue()
tripleValue()
squareValue()
```

we can create one generic method:

```java
applyFunction()
```

and pass different functions dynamically.

---

# 3. Returning a Function from Another Function

A function can return another function.

This is a powerful Functional Programming concept.

---

## Example

```java
import java.util.function.Function;

public class ReturnFunctionExample
{
    public static Function<Integer,Integer>
    createMultiplier(int multiplier)
    {
        return number -> number * multiplier;
    }

    public static void main(String[] args)
    {
        Function<Integer,Integer> triple =
                createMultiplier(3);

        System.out.println(
                triple.apply(5));
    }
}
```

Output:

```text
15
```

---

# How It Works

### Method Call

```java
createMultiplier(3)
```

Returns:

```java
number -> number * 3
```

---

### Store Returned Function

```java
Function<Integer,Integer> triple
```

---

### Execute Function

```java
triple.apply(5);
```

Result:

```text
15
```

---

# Visual Representation

```text
createMultiplier(3)
          |
          v
number -> number * 3
          |
          v
triple.apply(5)
          |
          v
15
```

# What is Function<T,R>?

The examples use:

```java
Function<T,R>
```

which is a Functional Interface provided by Java.

Package:

```java
java.util.function
```

---

## Syntax

```java
Function<InputType, OutputType>
```

Example:

```java
Function<Integer,Integer>
```

Meaning:

```text
Input  -> Integer
Output -> Integer
```

---

# What is a Functional Interface?

A Functional Interface contains exactly one abstract method.

Example:

```java
@FunctionalInterface
interface Calculator
{
    int calculate(int a, int b);
}
```

Lambda Expressions are used to implement Functional Interfaces.

---

# Lambda Expression Recap

Traditional Method:

```java
public int square(int number)
{
    return number * number;
}
```

Lambda Version:

```java
number -> number * number
```

Shorter and cleaner.

# Advantages of First-Class Functions

## 1. Reusable Code

Write a method once.

Pass different functions.

Example:

```java
applyFunction()
```

can work with:

* Square
* Double
* Triple
* Cube

without modification.

---

## 2. Dynamic Behavior

Behavior can be changed at runtime.

```java
applyFunction(square, 5);

applyFunction(doubleValue, 5);

applyFunction(cube, 5);
```

---

## 3. Less Code

Reduces duplicate methods.

---

## 4. Better Readability

Code becomes more expressive.

---

## 5. Supports Functional Programming

Required for:

* Streams
* Lambdas
* Method References

# Real-World Examples

## Stream API

```java
numbers.stream()
       .map(number -> number * number)
       .forEach(System.out::println);
```

---

## Sorting

```java
employees.sort(
    (e1, e2) ->
        e1.getName()
          .compareTo(e2.getName()));
```

---

## Filtering

```java
numbers.stream()
       .filter(n -> n % 2 == 0);
```

These operations rely heavily on First-Class Functions.

# Summary

* Functions as First-Class Objects are a core Functional Programming concept.
* Java supports this using:

  * Lambda Expressions
  * Functional Interfaces
* First-Class Functions can:

  * Be assigned to variables
  * Be passed as arguments
  * Be returned from functions
* The `Function<T,R>` interface is commonly used.
* First-Class Functions make code:

  * Reusable
  * Flexible
  * Cleaner
  * More maintainable

Functions are treated like data, which allows Java developers to write powerful and expressive Functional Programming code.
