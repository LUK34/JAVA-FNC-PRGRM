# Pure Functions in Functional Programming

## Introduction

Pure Functions are one of the most important concepts in Functional Programming.

A Pure Function is a function that:

1. Always produces the same output for the same input.
2. Has no side effects.

Pure Functions are predictable, easy to test, and safe to use in multi-threaded applications.

---

# What is a Pure Function?

A Pure Function follows two rules.

## Rule 1: Same Input → Same Output

If you call a Pure Function multiple times with the same input, it will always return the same result.

Example:

```java
applyDiscount(1000, 10)
```

Output:

```text
900
```

No matter how many times you call it:

```java
applyDiscount(1000, 10)
applyDiscount(1000, 10)
applyDiscount(1000, 10)
```

The result will always be:

```text
900
900
900
```

---

## Rule 2: No Side Effects

A Pure Function must not:

* Modify global variables
* Modify instance variables
* Modify external data
* Depend on external state

The function should depend only on its input parameters.

---

# Real-Life Analogy

Think of a vending machine.

If you insert:

```text
₹10
```

and choose:

```text
Chips
```

you will always receive the same product.

The vending machine does not change its behavior based on hidden variables.

This is similar to a Pure Function.

```text
Input → Processing → Output
```

Same input always produces the same output.

# Example of a Pure Function

## Discount Calculation

```java
public class PureFunctionExample
{
    public static double applyDiscount(double price, double discount)
    {
        return price - (price * discount / 100);
    }

    public static void main(String[] args)
    {
        System.out.println(applyDiscount(1000, 10));
        System.out.println(applyDiscount(1000, 10));
    }
}
```

Output:

```text
900.0
900.0
```

---

## Why is it Pure?

The method:

```java
applyDiscount()
```

* Depends only on its parameters.
* Does not use external variables.
* Produces the same output for the same input.
* Has no side effects.

Therefore it is a Pure Function.

---

# Example of an Impure Function

```java
public class ImpureFunctionExample
{
    static double discount = 10;

    public static double applyDiscount(double price)
    {
        return price - (price * discount / 100);
    }

    public static void main(String[] args)
    {
        System.out.println(applyDiscount(1000));

        discount = 20;

        System.out.println(applyDiscount(1000));
    }
}
```

Output:

```text
900.0
800.0
```

---

# Why is it Impure?

The method:

```java
applyDiscount()
```

depends on:

```java
discount
```

which is a global variable.

When the value changes:

```java
discount = 10
```

to

```java
discount = 20
```

the same input:

```java
1000
```

produces different outputs.

Therefore it is not a Pure Function.

# Pure Function Example #2

## Count Characters

```java
public static int countCharacters(String input)
{
    return input.length();
}
```

Usage:

```java
System.out.println(countCharacters("Hello"));
System.out.println(countCharacters("Hello"));
```

Output:

```text
5
5
```

---

## Why is it Pure?

The method:

```java
countCharacters()
```

* Depends only on the input parameter.
* Does not modify external variables.
* Always returns the same result.

Therefore it is a Pure Function.

---

# Impure Version

```java
public class ImpureCharacterCounter
{
    static int totalCharacters = 0;

    public static int countCharacters(String input)
    {
        totalCharacters += input.length();

        return totalCharacters;
    }

    public static void main(String[] args)
    {
        System.out.println(countCharacters("Hello"));
        System.out.println(countCharacters("Hello"));
    }
}
```

Output:

```text
5
10
```

---

# Why is it Impure?

The method modifies:

```java
totalCharacters
```

which is an external state variable.

The same input:

```java
"Hello"
```

produces different outputs.

Therefore it is not a Pure Function.

---

# Pure Function vs Impure Function

| Feature                  | Pure Function | Impure Function |
| ------------------------ | ------------- | --------------- |
| Same Input = Same Output | Yes           | Not Always      |
| Uses External Variables  | No            | Yes             |
| Modifies State           | No            | Yes             |
| Side Effects             | No            | Yes             |
| Easy to Test             | Yes           | No              |
| Thread Safe              | Generally Yes | Risky           |
| Predictable              | Yes           | No              |

---

# What are Side Effects?

A side effect occurs when a method changes something outside its scope.

Examples:

### Modifying Global Variables

```java
counter++;
```

---

### Writing to a File

```java
fileWriter.write("Hello");
```

---

### Updating a Database

```java
repository.save(employee);
```

---

### Sending an Email

```java
emailService.send();
```

---

These operations affect external systems and are considered side effects.

---

# Advantages of Pure Functions

## 1. Predictable Behavior

Pure Functions always produce the same result.

```java
countCharacters("Java")
```

will always return:

```text
4
```

This makes the code easier to understand.

---

## 2. No Side Effects

Pure Functions do not change external data.

This eliminates many bugs caused by shared state.

---

## 3. Easy to Test

Testing becomes very simple.

Example:

```java
assertEquals(900,
             applyDiscount(1000,10));
```

No additional setup is required.

---

## 4. Easier Debugging

Since there are no hidden state changes:

* Fewer bugs
* Easier troubleshooting
* Easier maintenance

---

## 5. Supports Parallel Processing

Pure Functions do not modify shared data.

Therefore multiple threads can safely execute them simultaneously.

```java
numbers.parallelStream()
```

works much better when using Pure Functions.

---

# When Should You Use Pure Functions?

Use Pure Functions when:

* Writing reusable code
* Writing testable code
* Working with Stream API
* Using Lambda Expressions
* Using Method References
* Performing data transformations
* Developing multi-threaded applications

---

# Functional Programming and Pure Functions

Functional Programming encourages writing Pure Functions whenever possible.

Common Functional Programming features include:

## Lambda Expressions

```java
x -> x * 2
```

---

## Method References

```java
System.out::println
```

---

## Stream API

```java
numbers.stream()
       .filter(n -> n % 2 == 0)
       .count();
```

These operations work best when Pure Functions are used.

---

# Summary

* Pure Functions are a core concept of Functional Programming.
* A Pure Function follows two rules:

  1. Same input always produces the same output.
  2. No side effects.
* Pure Functions do not modify external variables.
* Pure Functions are:

  * Predictable
  * Reusable
  * Easy to test
  * Easy to debug
  * Better for parallel processing
* Impure Functions depend on or modify external state.
* Functional Programming encourages the use of Pure Functions whenever possible.

Writing Pure Functions leads to cleaner, safer, and more maintainable Java code.
