# Java HashSet Class Overview

## Introduction

`HashSet` is a part of the Java Collections Framework and is used to store a collection of **unique elements**.

The `HashSet` class implements the `Set` interface and is located in the `java.util` package.

```java
import java.util.HashSet;
```

---

# Key Points About HashSet

## 1. HashSet Implements Set Interface

The `HashSet` class implements the `Set` interface.

```java
HashSet<String> languages = new HashSet<>();
```

Since it implements the `Set` interface, it stores only unique elements.

---

## 2. Stores Unique Elements

A `HashSet` does not allow duplicate values.

If you try to add a duplicate element, HashSet simply ignores it.

```java
HashSet<String> fruits = new HashSet<>();

fruits.add("Apple");
fruits.add("Banana");
fruits.add("Apple");
```

Output:

```text
[Apple, Banana]
```

The second `"Apple"` is ignored.

---

## 3. Allows One Null Value

HashSet allows only one `null` value.

```java
HashSet<String> fruits = new HashSet<>();

fruits.add("Apple");
fruits.add(null);
fruits.add(null);
```

Output:

```text
[Apple, null]
```

The second `null` value is ignored.

---

## 4. Unordered Collection

HashSet does **not maintain insertion order**.

```java
HashSet<String> fruits = new HashSet<>();

fruits.add("Apple");
fruits.add("Banana");
fruits.add("Mango");
```

Output may be:

```text
[Mango, Apple, Banana]
```

or

```text
[Banana, Mango, Apple]
```

The order is not guaranteed.

### Comparison with ArrayList

| Collection | Maintains Insertion Order |
| ---------- | ------------------------- |
| ArrayList  | Yes                       |
| LinkedList | Yes                       |
| HashSet    | No                        |

---

## 5. Internally Uses HashMap

HashSet internally uses a HashMap to store elements.

When an element is added:

* The element is stored as a **key**
* A dummy object is stored as the **value**

Conceptually:

```text
Apple  -> Dummy Object
Banana -> Dummy Object
Mango  -> Dummy Object
```

This is how HashSet ensures uniqueness.

---

## 6. Dynamic Size

HashSet automatically grows and shrinks as elements are added or removed.

```java
HashSet<String> fruits = new HashSet<>();

fruits.add("Apple");
fruits.add("Banana");

fruits.remove("Apple");
```

No size declaration is required.

---

## 7. Not Thread-Safe

HashSet is not synchronized.

If multiple threads access a HashSet simultaneously, unpredictable results may occur.

```java
HashSet<String> fruits = new HashSet<>();
```

For multi-threaded applications, synchronization must be handled explicitly.

---

## 8. Primitive Types Are Not Allowed

HashSet uses Generics.

Generics do not support primitive types directly.

❌ Invalid

```java
HashSet<int> numbers = new HashSet<>();
```

✅ Valid

```java
HashSet<Integer> numbers = new HashSet<>();
```

Use Wrapper Classes:

| Primitive | Wrapper Class |
| --------- | ------------- |
| int       | Integer       |
| long      | Long          |
| double    | Double        |
| float     | Float         |
| char      | Character     |
| boolean   | Boolean       |

# Common HashSet Methods

## add()

Adds an element.

```java
fruits.add("Apple");
```

---

## remove()

Removes an element.

```java
fruits.remove("Apple");
```

---

## contains()

Checks whether an element exists.

```java
fruits.contains("Apple");
```

Returns:

```text
true
```

or

```text
false
```

---

## size()

Returns the number of elements.

```java
fruits.size();
```

---

## isEmpty()

Checks whether the set is empty.

```java
fruits.isEmpty();
```

---

## clear()

Removes all elements.

```java
fruits.clear();
```

---

# Advantages of HashSet

* Stores only unique elements
* Fast insertion
* Fast deletion
* Fast searching
* Allows one null value
* Automatically resizes

---

# Disadvantages of HashSet

* Does not maintain insertion order
* Not thread-safe
* No index-based access
* Cannot store duplicate values

---

# HashSet vs ArrayList

| Feature            | ArrayList        | HashSet        |
| ------------------ | ---------------- | -------------- |
| Duplicate Elements | Allowed          | Not Allowed    |
| Insertion Order    | Maintained       | Not Maintained |
| Null Values        | Multiple Allowed | One Allowed    |
| Index Access       | Yes              | No             |
| Search Performance | Slower           | Faster         |
| Internal Structure | Dynamic Array    | HashMap        |

---

# When to Use HashSet

Use HashSet when:

* You need unique elements.
* Duplicate values must be avoided.
* Fast searching is important.
* Element order does not matter.

Examples:

* Unique employee IDs
* Unique email addresses
* Unique usernames
* Unique product codes

---

# Summary

* `HashSet` is part of the Java Collections Framework.
* Located in the `java.util` package.
* Implements the `Set` interface.
* Stores only unique elements.
* Duplicate elements are ignored.
* Allows one `null` value.
* Does not maintain insertion order.
* Internally uses `HashMap`.
* Not thread-safe.
* Does not support primitive data types directly.
* Provides fast insertion, deletion, and searching operations.

HashSet is the preferred collection when uniqueness is more important than maintaining the order of elements.
