# Java LinkedList Class Overview

## Introduction

`LinkedList` is a part of the Java Collections Framework and is used to store a collection of elements in a dynamic manner.

Unlike arrays, a `LinkedList` can grow and shrink automatically as elements are added or removed.

The `LinkedList` class is located in the `java.util` package.

```java
import java.util.LinkedList;
```

---

# Key Points About LinkedList

## 1. LinkedList Implements List and Deque Interfaces

The `LinkedList` class implements both:

* `List` Interface
* `Deque` Interface

Because of this, a `LinkedList` can be used as:

* List
* Queue
* Deque
* Stack

```java
LinkedList<String> languages = new LinkedList<>();
```

---

## 2. Doubly Linked List Implementation

Java `LinkedList` is implemented as a **Doubly Linked List**.

Each element (node) contains:

* Data
* Reference to Previous Node
* Reference to Next Node

```text
null <- [Java] <-> [Python] <-> [C++] -> null
```

This allows traversal in both directions.

---

## 3. Maintains Insertion Order

LinkedList preserves the order in which elements are inserted.

```java
LinkedList<String> list = new LinkedList<>();

list.add("Java");
list.add("Python");
list.add("C++");
```

Output:

```text
Java
Python
C++
```

Elements are retrieved in the same order they were inserted.

---

## 4. Dynamic Size

LinkedList can grow and shrink automatically.

No size needs to be specified during creation.

```java
LinkedList<String> list = new LinkedList<>();

list.add("Java");
list.add("Python");

list.remove("Java");
```

The list automatically adjusts its size.

---

## 5. Allows Duplicate Values

LinkedList can store duplicate elements.

```java
LinkedList<String> list = new LinkedList<>();

list.add("Java");
list.add("Java");
list.add("Python");
```

Output:

```text
[Java, Java, Python]
```

---

## 6. Allows Null Values

LinkedList can store null values.

```java
LinkedList<String> list = new LinkedList<>();

list.add("Java");
list.add(null);
list.add("Python");
```

Output:

```text
[Java, null, Python]
```

---

## 7. Can Be Used as Queue

Since LinkedList implements the `Deque` interface, it can be used as a Queue.

### FIFO (First In First Out)

```java
LinkedList<String> queue = new LinkedList<>();

queue.offer("A");
queue.offer("B");
queue.offer("C");

System.out.println(queue.poll());
```

Output:

```text
A
```

---

## 8. Can Be Used as Stack

LinkedList also supports Stack operations.

### LIFO (Last In First Out)

```java
LinkedList<String> stack = new LinkedList<>();

stack.push("A");
stack.push("B");
stack.push("C");

System.out.println(stack.pop());
```

Output:

```text
C
```

---

## 9. Primitive Types Are Not Allowed

LinkedList uses Generics.

Generics do not support primitive data types.

❌ Invalid

```java
LinkedList<int> numbers = new LinkedList<>();
```

✅ Valid

```java
LinkedList<Integer> numbers = new LinkedList<>();
```

Use Wrapper Classes:

| Primitive | Wrapper Class |
| --------- | ------------- |
| int       | Integer       |
| long      | Long          |
| double    | Double        |
| float     | Float         |
| char      | Character     |
| boolean   | Boolean       |

---

## 10. Not Thread-Safe

LinkedList is not synchronized.

If multiple threads access a LinkedList simultaneously, the result may become unpredictable.

```java
LinkedList<String> list = new LinkedList<>();
```

For multi-threaded applications, synchronization must be handled explicitly.

# Advantages of LinkedList

* Dynamic size
* Maintains insertion order
* Allows duplicates
* Allows null values
* Can be used as List, Queue, Deque, and Stack
* Efficient insertion and deletion operations

---

# Disadvantages of LinkedList

* More memory consumption than ArrayList
* Slower random access compared to ArrayList
* Not thread-safe

---

# ArrayList vs LinkedList

| Feature            | ArrayList     | LinkedList         |
| ------------------ | ------------- | ------------------ |
| Internal Structure | Dynamic Array | Doubly Linked List |
| Random Access      | Fast          | Slow               |
| Insertion          | Slower        | Faster             |
| Deletion           | Slower        | Faster             |
| Memory Usage       | Less          | More               |
| Maintains Order    | Yes           | Yes                |
| Allows Duplicates  | Yes           | Yes                |
| Allows Null Values | Yes           | Yes                |

---

# Summary

* `LinkedList` is part of the Java Collections Framework.
* Located in the `java.util` package.
* Implemented as a Doubly Linked List.
* Maintains insertion order.
* Supports dynamic resizing.
* Allows duplicate and null values.
* Implements both `List` and `Deque` interfaces.
* Can be used as a List, Queue, Deque, or Stack.
* Does not support primitive data types directly.
* Not thread-safe.

LinkedList is a good choice when your application performs frequent insertions and deletions of elements.
