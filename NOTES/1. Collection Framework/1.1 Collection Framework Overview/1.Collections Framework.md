# Java Collections Framework

## Introduction

The Java Collections Framework (JCF) is a unified architecture that provides ready-made data structures and algorithms for storing, managing, and processing data efficiently.

Instead of manually managing arrays, Java Collections provides dynamic, flexible, and optimized data structures that simplify application development.

The Collections Framework consists of:

* Interfaces
* Implementations (Classes)
* Utility Classes

These components work together to provide efficient data storage and manipulation capabilities.

```text

   └── Collection
         │
         ├── List

                       └── TreeMap
```

# 1. Iterable Interface

`Iterable` is the top-most interface in the Collections Framework hierarchy.

It represents a collection of elements that can be traversed one by one.

The main purpose of the `Iterable` interface is to provide support for iteration using:

* Enhanced for-loop (`for-each`)
* Iterator

Example:

```java
for(String name : names)
{
    System.out.println(name);
}
```

---

# 2. Collection Interface

The `Collection` interface is the root interface of the Java Collections Framework.

A collection represents a group of objects known as elements.

This interface provides common operations such as:

* add()
* remove()
* contains()
* size()
* isEmpty()

All major collection interfaces inherit these common methods.

---

# 3. List Interface

The `List` interface extends the `Collection` interface.

A List represents an ordered collection of elements.

## Characteristics

* Maintains insertion order
* Allows duplicate elements
* Supports index-based access

## Common Implementations

### ArrayList

* Dynamic array implementation
* Fast random access
* Best for frequent reads

### LinkedList

* Doubly linked list implementation
* Efficient insertion and deletion
* Slower random access

### Vector (Legacy)

* Synchronized implementation
* Rarely used in modern applications

Example:

```java
List<String> names = new ArrayList<>();

names.add("Luke");
names.add("John");
names.add("Luke");
```

Duplicates are allowed.

---

# 4. Set Interface

The `Set` interface extends the `Collection` interface.

A Set stores unique elements.

## Characteristics

* No duplicate elements
* No index-based access
* Used when uniqueness is required

## Common Implementations

### HashSet

* Fastest implementation
* Does not maintain insertion order

### LinkedHashSet

* Maintains insertion order
* Slightly slower than HashSet

### TreeSet

* Stores elements in sorted order
* Implements SortedSet

Example:

```java
Set<String> names = new HashSet<>();

names.add("Luke");
names.add("John");
names.add("Luke");
```

Output:

```text
Luke
John
```

Duplicate value is ignored.

---

# 5. SortedSet Interface

`SortedSet` extends the `Set` interface.

It maintains elements in ascending sorted order.

## Implementation

### TreeSet

Example:

```java
SortedSet<Integer> numbers = new TreeSet<>();

numbers.add(30);
numbers.add(10);
numbers.add(20);
```

Output:

```text
10
20
30
```

---

# 6. Queue Interface

The `Queue` interface extends the `Collection` interface.

A Queue is typically used for processing elements in First-In-First-Out (FIFO) order.

## Characteristics

* First element inserted is processed first
* Commonly used in task scheduling and messaging systems

## Common Implementations

### PriorityQueue

Elements are processed based on priority.

### LinkedList

Can also be used as a Queue implementation.

Example:

```java
Queue<String> queue = new LinkedList<>();

queue.offer("A");
queue.offer("B");

System.out.println(queue.poll());
```

Output:

```text
A
```

---

# 7. Deque Interface

`Deque` (Double Ended Queue) extends the `Queue` interface.

It allows insertion and deletion from both ends.

## Common Implementations

### ArrayDeque

High-performance implementation.

### LinkedList

Can also act as a Deque.

Example:

```java
Deque<String> deque = new ArrayDeque<>();

deque.addFirst("A");
deque.addLast("B");
```

---

# 8. BlockingQueue Interface

`BlockingQueue` extends the `Queue` interface.

It is primarily used in multi-threaded applications.

## Common Implementations

* ArrayBlockingQueue
* LinkedBlockingQueue

These queues automatically block producer or consumer threads when necessary.

Commonly used in:

* Producer-Consumer Pattern
* Thread Pools
* Concurrent Applications

---

# 9. Map Interface

The `Map` interface is part of the Collections Framework but does NOT extend the `Collection` interface.

A Map stores data in key-value pairs.

## Characteristics

* Keys must be unique
* Values can be duplicated
* Fast lookup by key

## Common Implementations

### HashMap

* Most commonly used implementation
* Does not maintain insertion order

### LinkedHashMap

* Maintains insertion order

### Hashtable (Legacy)

* Synchronized
* Rarely used in modern development

Example:

```java
Map<Integer, String> employees = new HashMap<>();

employees.put(1001, "Luke");
employees.put(1002, "John");
```

---

# 10. SortedMap Interface

`SortedMap` extends the `Map` interface.

It stores keys in ascending sorted order.

## Implementation

### TreeMap

Example:

```java
SortedMap<Integer, String> map = new TreeMap<>();

map.put(3, "C");
map.put(1, "A");
map.put(2, "B");
```

Output:

```text
1=A
2=B
3=C
```

# Legacy Collection Classes

The following classes are considered legacy classes:

* Vector
* Hashtable
* Stack

These classes were introduced before the Collections Framework was redesigned.

In modern Java development, they are rarely used and are generally replaced by:

| Legacy Class | Preferred Alternative |
| ------------ | --------------------- |
| Vector       | ArrayList             |
| Hashtable    | HashMap               |
| Stack        | Deque / ArrayDeque    |

# Most Commonly Used Collection Classes in Real Projects

The following collection implementations are heavily used in modern Java and Spring Boot applications:

### List Implementations

* ArrayList
* LinkedList

### Set Implementations

* HashSet
* LinkedHashSet
* TreeSet

### Map Implementations

* HashMap
* LinkedHashMap
* TreeMap

These collection classes form the foundation of day-to-day Java development and are frequently used in enterprise applications, Spring Boot projects, REST APIs, and microservices.

# Key Takeaways

* `Iterable` provides iteration capability.
* `Collection` is the root interface of the Collections Framework.
* `List` maintains order and allows duplicates.
* `Set` stores unique elements.
* `TreeSet` stores elements in sorted order.
* `Queue` follows FIFO processing.
* `Deque` supports insertion/removal from both ends.
* `BlockingQueue` is used in concurrent programming.
* `Map` stores key-value pairs.
* `TreeMap` stores keys in sorted order.
* Legacy classes such as Vector, Hashtable, and Stack are rarely used in modern applications.
