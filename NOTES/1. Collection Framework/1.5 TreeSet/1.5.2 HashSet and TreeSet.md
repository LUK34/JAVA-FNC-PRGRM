# HashSet vs TreeSet in Java

## Introduction

`HashSet` and `TreeSet` are two popular implementations of the `Set` interface in Java.

Both collections:

* Store unique elements
* Do not allow duplicate values
* Are part of the Java Collections Framework

However, they differ in how they store, organize, and retrieve elements.

Understanding these differences helps developers choose the right collection based on application requirements.

---

# HashSet Overview

`HashSet` is a collection that stores unique elements without maintaining any ordering.

### Characteristics

* No ordering of elements
* Allows one null value
* Uses HashMap internally
* Faster performance for basic operations
* Implements Set interface

### Example

```java id="c9n5h1"
Set<String> languages = new HashSet<>();

languages.add("Java");
languages.add("Python");
languages.add("C++");

System.out.println(languages);
```

### Possible Output

```text id="rn0h7l"
[Python, Java, C++]
```

The order is not guaranteed.

---

# TreeSet Overview

`TreeSet` stores unique elements in a sorted order.

### Characteristics

* Maintains sorted order
* Does not allow null values
* Uses TreeMap internally
* Supports navigation methods
* Implements NavigableSet interface

### Example

```java id="3uphnn"
Set<String> languages = new TreeSet<>();

languages.add("Java");
languages.add("Python");
languages.add("C++");

System.out.println(languages);
```

### Output

```text id="18g2wz"
[C++, Java, Python]
```

The elements are automatically sorted alphabetically.

---

# Difference Between HashSet and TreeSet

| Feature                 | HashSet                | TreeSet                |
| ----------------------- | ---------------------- | ---------------------- |
| Ordering                | No ordering maintained | Maintains sorted order |
| Duplicate Elements      | Not allowed            | Not allowed            |
| Null Values             | One null allowed       | Null not allowed       |
| Internal Data Structure | HashMap                | TreeMap                |
| Interface Implemented   | Set                    | NavigableSet           |
| Sorting Support         | No                     | Yes                    |
| Navigation Methods      | No                     | Yes                    |
| Performance             | Faster                 | Slightly slower        |
| Natural Ordering        | No                     | Yes                    |
| Custom Comparator       | No                     | Yes                    |

---

# 1. Ordering of Elements

## HashSet

HashSet does not maintain any order.

```java id="39smf9"
HashSet<Integer> numbers = new HashSet<>();

numbers.add(30);
numbers.add(10);
numbers.add(20);

System.out.println(numbers);
```

### Possible Output

```text id="d5syvq"
[20, 10, 30]
```

The order may vary.

---

## TreeSet

TreeSet automatically sorts elements.

```java id="e96xqh"
TreeSet<Integer> numbers = new TreeSet<>();

numbers.add(30);
numbers.add(10);
numbers.add(20);

System.out.println(numbers);
```

### Output

```text id="j9cz5f"
[10, 20, 30]
```

---

# 2. Null Values

## HashSet

HashSet allows one null value.

```java id="ovlw2v"
HashSet<String> set = new HashSet<>();

set.add("Java");
set.add(null);

System.out.println(set);
```

### Output

```text id="nlm8qn"
[null, Java]
```

---

## TreeSet

TreeSet does not allow null values.

```java id="ab0wj4"
TreeSet<String> set = new TreeSet<>();

set.add("Java");
set.add(null);
```

### Output

```text id="vygvzl"
Exception in thread "main"
java.lang.NullPointerException
```

TreeSet compares elements to maintain sorting, and null cannot be compared.

---

# 3. Internal Implementation

## HashSet

Internally uses:

```text id="c4yokd"
HashMap
```

Each element becomes a key in an internal HashMap.

---

## TreeSet

Internally uses:

```text id="gdhwzd"
TreeMap
```

Each element becomes a key in an internal TreeMap.

Because TreeMap keeps keys sorted, TreeSet also remains sorted.

---

# 4. Interface Hierarchy

## HashSet Hierarchy

```text id="x4i0tv"
Iterable
    ↑
Collection
    ↑
Set
    ↑
HashSet
```

HashSet implements the `Set` interface.

---

## TreeSet Hierarchy

```text id="mxz90j"
Iterable
    ↑
Collection
    ↑
Set
    ↑
SortedSet
    ↑
NavigableSet
    ↑
TreeSet
```

TreeSet implements `NavigableSet`, which extends `SortedSet`.

---

# 5. Navigation Support

TreeSet provides navigation methods.

### Example

```java id="1ckgvy"
TreeSet<Integer> numbers = new TreeSet<>();

numbers.add(10);
numbers.add(20);
numbers.add(30);
numbers.add(40);

System.out.println(numbers.higher(20));
System.out.println(numbers.lower(20));
```

### Output

```text id="zj8i53"
30
10
```

HashSet does not provide these methods.

---

# 6. Custom Comparator Support

TreeSet allows custom sorting.

### Reverse Order Example

```java id="0vc02q"
TreeSet<Integer> numbers =
        new TreeSet<>(Comparator.reverseOrder());

numbers.add(10);
numbers.add(30);
numbers.add(20);

System.out.println(numbers);
```

### Output

```text id="40mcbh"
[30, 20, 10]
```

HashSet does not support sorting or comparators.

---

# Performance Comparison

| Operation  | HashSet | TreeSet  |
| ---------- | ------- | -------- |
| add()      | O(1)    | O(log n) |
| remove()   | O(1)    | O(log n) |
| contains() | O(1)    | O(log n) |

HashSet is generally faster because it uses hashing.

TreeSet is slightly slower because it maintains sorted order.

---

# When to Use HashSet

Use HashSet when:

* You need unique elements
* Ordering is not important
* High performance is required
* Fast searching is required

### Example Use Cases

* Removing duplicates
* Lookup operations
* Membership testing

---

# When to Use TreeSet

Use TreeSet when:

* You need unique elements
* Elements must remain sorted
* Navigation methods are required
* Range-based operations are needed

### Example Use Cases

* Leaderboards
* Rankings
* Sorted lists
* Navigation-based searching

---

# Key Points to Remember

1. Both HashSet and TreeSet store unique elements.
2. HashSet does not maintain ordering.
3. TreeSet automatically sorts elements.
4. HashSet allows one null value.
5. TreeSet does not allow null values.
6. HashSet uses HashMap internally.
7. TreeSet uses TreeMap internally.
8. HashSet is faster than TreeSet.
9. TreeSet provides navigation methods.
10. TreeSet supports custom comparators.

---

# Summary

HashSet and TreeSet are both implementations of the Set interface used to store unique elements. HashSet focuses on performance and does not maintain ordering, while TreeSet focuses on maintaining elements in sorted order and provides powerful navigation capabilities. The choice between them depends on whether sorting or performance is more important for your application.
