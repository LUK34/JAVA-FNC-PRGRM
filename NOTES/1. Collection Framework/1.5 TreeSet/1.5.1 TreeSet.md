# TreeSet in Java

## Introduction

`TreeSet` is a part of the Java Collections Framework and is used to store a collection of unique elements in a sorted order.

Unlike `HashSet`, which does not maintain any ordering, `TreeSet` automatically sorts its elements either according to their natural ordering or according to a custom comparator provided at the time of creation.

`TreeSet` is commonly used when you need:

* Unique elements
* Sorted elements
* Fast navigation through sorted data

---

# TreeSet Class Hierarchy

```text
Iterable
    ↑
Collection
    ↑
Set
    ↑
SortedSet
    ↑
NavigableSet
    ↑
TreeSet
```

`TreeSet` implements the `NavigableSet` interface, which extends the `SortedSet` interface.

As a result, `TreeSet` provides:

* Sorted data storage
* Navigation methods
* Unique elements

---

# Important Characteristics of TreeSet

## 1. TreeSet Maintains Sorted Order

Elements are automatically sorted when they are inserted.

### Example

```java
TreeSet<Integer> numbers = new TreeSet<>();

numbers.add(30);
numbers.add(10);
numbers.add(20);

System.out.println(numbers);
```

### Output

```text
[10, 20, 30]
```

---

## 2. TreeSet Uses TreeMap Internally

Internally, `TreeSet` is backed by a `TreeMap`.

```text
TreeSet
    ↓
TreeMap
```

The elements of the TreeSet are stored as keys inside the TreeMap.

Because TreeMap maintains sorted keys, TreeSet automatically maintains sorted elements.

---

## 3. TreeSet Does Not Allow Duplicate Elements

Duplicate values are ignored.

### Example

```java
TreeSet<String> languages = new TreeSet<>();

languages.add("Java");
languages.add("Python");
languages.add("Java");

System.out.println(languages);
```

### Output

```text
[Java, Python]
```

---

## 4. TreeSet Does Not Allow Null Values

Adding a null value causes a `NullPointerException`.

### Example

```java
TreeSet<String> languages = new TreeSet<>();

languages.add("Java");
languages.add(null);
```

### Output

```text
Exception in thread "main"
java.lang.NullPointerException
```

TreeSet must compare elements to keep them sorted, and `null` cannot be compared with other values.

---

## 5. Natural Ordering

By default, TreeSet sorts elements using their natural ordering.

### Numbers

```java
TreeSet<Integer> numbers = new TreeSet<>();

numbers.add(50);
numbers.add(10);
numbers.add(30);

System.out.println(numbers);
```

### Output

```text
[10, 30, 50]
```

### Strings

```java
TreeSet<String> names = new TreeSet<>();

names.add("John");
names.add("Alex");
names.add("David");

System.out.println(names);
```

### Output

```text
[Alex, David, John]
```

Strings are sorted alphabetically.

---

## 6. Custom Ordering Using Comparator

A custom comparator can be supplied when creating a TreeSet.

### Reverse Order Example

```java
TreeSet<Integer> numbers =
        new TreeSet<>(Comparator.reverseOrder());

numbers.add(10);
numbers.add(30);
numbers.add(20);

System.out.println(numbers);
```

### Output

```text
[30, 20, 10]
```

---

## 7. TreeSet Is Not Thread Safe

TreeSet is not synchronized.

If multiple threads access and modify a TreeSet simultaneously, unexpected results may occur.

### Synchronizing TreeSet

```java
Set<Integer> synchronizedSet =
        Collections.synchronizedSet(new TreeSet<>());
```

# Common TreeSet Methods

| Method             | Description                           |
| ------------------ | ------------------------------------- |
| add(E e)           | Adds an element                       |
| remove(Object o)   | Removes an element                    |
| contains(Object o) | Checks whether element exists         |
| size()             | Returns number of elements            |
| isEmpty()          | Checks whether set is empty           |
| clear()            | Removes all elements                  |
| first()            | Returns first element                 |
| last()             | Returns last element                  |
| higher(E e)        | Returns next higher element           |
| lower(E e)         | Returns next lower element            |
| ceiling(E e)       | Returns equal or next greater element |
| floor(E e)         | Returns equal or next smaller element |

---

# Navigation Methods Example

```java
import java.util.TreeSet;

public class TreeSetNavigationExample {

    public static void main(String[] args) {

        TreeSet<Integer> numbers = new TreeSet<>();

        numbers.add(10);
        numbers.add(20);
        numbers.add(30);
        numbers.add(40);
        numbers.add(50);

        System.out.println("First  : " + numbers.first());
        System.out.println("Last   : " + numbers.last());
        System.out.println("Higher : " + numbers.higher(20));
        System.out.println("Lower  : " + numbers.lower(20));
        System.out.println("Ceiling: " + numbers.ceiling(25));
        System.out.println("Floor  : " + numbers.floor(25));
    }
}
```

### Output

```text
First  : 10
Last   : 50
Higher : 30
Lower  : 10
Ceiling: 30
Floor  : 20
```

# Complete Example

```java
import java.util.TreeSet;

public class CreateTreeSetExample {

    public static void main(String[] args) {

        TreeSet<String> languages = new TreeSet<>();

        languages.add("Java");
        languages.add("Python");
        languages.add("Go");
        languages.add("Java");
        languages.add("Kotlin");

        System.out.println("Languages: " + languages);

        System.out.println("First Element : " + languages.first());
        System.out.println("Last Element  : " + languages.last());

        languages.remove("Go");

        System.out.println("After Remove  : " + languages);
    }
}
```

### Output

```text
Languages: [Go, Java, Kotlin, Python]
First Element : Go
Last Element  : Python
After Remove  : [Java, Kotlin, Python]
```

# TreeSet vs HashSet

| Feature            | HashSet          | TreeSet          |
| ------------------ | ---------------- | ---------------- |
| Ordering           | No ordering      | Sorted order     |
| Duplicate Elements | Not allowed      | Not allowed      |
| Null Values        | One null allowed | Null not allowed |
| Internal Structure | HashMap          | TreeMap          |
| Performance        | Faster           | Slightly slower  |
| Navigation Methods | Not available    | Available        |

---

# Key Points to Remember

1. TreeSet is part of the Java Collections Framework.
2. TreeSet implements the NavigableSet interface.
3. NavigableSet extends SortedSet.
4. TreeSet internally uses TreeMap.
5. Elements are automatically sorted.
6. Duplicate elements are not allowed.
7. Null values are not allowed.
8. Elements can be sorted using natural ordering or a custom comparator.
9. TreeSet provides navigation methods such as higher(), lower(), first(), and last().
10. TreeSet is not thread-safe.

---

# Summary

TreeSet is a powerful collection class that stores unique elements in a sorted order. It is backed by a TreeMap and provides additional navigation capabilities through the NavigableSet interface. TreeSet is ideal when both uniqueness and sorting are required.
