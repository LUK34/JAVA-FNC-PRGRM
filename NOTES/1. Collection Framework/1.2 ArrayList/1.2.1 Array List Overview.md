# Java ArrayList - Overview and Key Concepts

## Introduction

`ArrayList` is one of the most commonly used collection classes in Java.

It is part of the Java Collections Framework and provides a dynamic way to store and manage a collection of elements.

Unlike traditional arrays, an `ArrayList` can grow and shrink automatically based on the number of elements it contains.

---

# What is ArrayList?

`ArrayList` is a class that:

* Implements the `List` interface
* Belongs to the `java.util` package
* Stores elements in insertion order
* Allows duplicate values
* Allows null values
* Provides fast access using indexes

Example:

```java
import java.util.ArrayList;
import java.util.List;

List<String> names = new ArrayList<>();
```

---

# Array vs ArrayList

| Feature           | Array   | ArrayList            |
| ----------------- | ------- | -------------------- |
| Size              | Fixed   | Dynamic              |
| Can Grow          | No      | Yes                  |
| Can Shrink        | No      | Yes                  |
| Supports Generics | No      | Yes                  |
| Built-in Methods  | Limited | Many Utility Methods |

### Array Example

```java
String[] names = new String[3];
```

The size is fixed to 3.

---

### ArrayList Example

```java
List<String> names = new ArrayList<>();

names.add("Luke");
names.add("John");
names.add("David");
names.add("Aquib");
```

No size needs to be specified.

The ArrayList automatically grows when new elements are added.

---

# ArrayList is a Resizable Array

An `ArrayList` is often called a **Dynamic Array** or **Resizable Array**.

This means:

* It automatically increases its capacity when new elements are added.
* It automatically reduces memory usage when elements are removed.
* Developers do not need to manually manage the size.

Example:

```java
List<Integer> numbers = new ArrayList<>();

for(int i = 1; i <= 1000; i++)
{
    numbers.add(i);
}
```

The ArrayList automatically expands to accommodate all 1000 elements.

---

# Internal Working of ArrayList

Although it behaves differently from an array, internally `ArrayList` uses an array to store elements.

```text
ArrayList
      ↓
Internal Array
      ↓
[Element0, Element1, Element2, Element3]
```

Because it uses an internal array, ArrayList provides fast access to elements using indexes.

---

# Index-Based Access

ArrayList allows elements to be accessed using their index positions.

Just like arrays, indexing starts from **0**.

Example:

```java
List<String> names = new ArrayList<>();

names.add("Luke");
names.add("John");
names.add("David");
```

Memory Representation:

```text
Index      Value
-----      -----
0          Luke
1          John
2          David
```

Accessing elements:

```java
System.out.println(names.get(0));
System.out.println(names.get(1));
```

Output:

```text
Luke
John
```

---

# ArrayList Allows Duplicate Values

Unlike a Set, an ArrayList allows duplicate elements.

Example:

```java
List<String> names = new ArrayList<>();

names.add("Luke");
names.add("Luke");
names.add("John");
```

Output:

```text
Luke
Luke
John
```

Duplicates are preserved.

---

# ArrayList Allows Null Values

ArrayList can store null values.

Example:

```java
List<String> names = new ArrayList<>();

names.add("Luke");
names.add(null);
names.add("John");
```

Output:

```text
Luke
null
John
```

---

# ArrayList Maintains Insertion Order

ArrayList preserves the order in which elements are inserted.

Example:

```java
List<String> names = new ArrayList<>();

names.add("Luke");
names.add("John");
names.add("David");
```

When iterating:

```java
for(String name : names)
{
    System.out.println(name);
}
```

Output:

```text
Luke
John
David
```

The insertion order is maintained.

---

# Primitive Types Are Not Allowed

ArrayList uses Generics.

Generics only support Objects and do not support primitive types.

### Invalid

```java
ArrayList<int> numbers = new ArrayList<>();
```

Compilation Error.

---

### Valid

```java
ArrayList<Integer> numbers = new ArrayList<>();
```

Use Wrapper Classes instead of primitive types.

| Primitive Type | Wrapper Class |
| -------------- | ------------- |
| int            | Integer       |
| long           | Long          |
| double         | Double        |
| float          | Float         |
| char           | Character     |
| boolean        | Boolean       |
| byte           | Byte          |
| short          | Short         |

Example:

```java
List<Integer> numbers = new ArrayList<>();

numbers.add(10);
numbers.add(20);
numbers.add(30);
```

---

# ArrayList is Not Synchronized

ArrayList is not thread-safe.

If multiple threads modify the same ArrayList simultaneously, unexpected results may occur.

Example:

```java
List<String> names = new ArrayList<>();
```

Multiple threads adding and removing elements at the same time can lead to inconsistent data.

---

# How to Make ArrayList Thread-Safe

Option 1:

```java
List<String> names =
    Collections.synchronizedList(new ArrayList<>());
```

Option 2:

Use `CopyOnWriteArrayList`.

```java
List<String> names =
    new CopyOnWriteArrayList<>();
```

# Key Takeaways

* ArrayList is part of the Java Collections Framework.
* It implements the List interface.
* It belongs to the `java.util` package.
* ArrayList is a dynamic and resizable array.
* It maintains insertion order.
* It allows duplicate values.
* It allows null values.
* Elements can be accessed using indexes.
* Primitive types are not supported directly.
* ArrayList is not synchronized by default.
* Internally, ArrayList uses an array for storage.

ArrayList is one of the most widely used collection classes in Java and is commonly used in enterprise applications, Spring Boot projects, REST APIs, and microservices.
